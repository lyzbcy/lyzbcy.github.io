## 后续维护时发现的注意事项

<details>
<summary><strong>本地测试功能正常，托管 GitHub 后功能缺失</strong></summary>

<p><strong>问题现象：</strong> 在本地 Jekyll 环境中测试时功能正常，但部署到 GitHub Pages 后某些功能（如动态效果）无法正常工作。</p>

<p><strong>原因分析：</strong> AI 在本地修改了 Chirpy 主题的静态资源（位于 <code>assets/lib</code> 子模块），但 GitHub Pages 构建时子模块可能未初始化，导致文件缺失。</p>

<p><strong>解决方案：</strong> 不要直接修改 Chirpy 主题的静态资源。如果需要修改，请将需要的文件复制到主仓库的相关目录（如 <code>assets/lib-custom/</code>），并更新所有引用路径。</p>

</details>

<details>
<summary><strong>在 HTML 标签（如 <code>&lt;details&gt;</code>、<code>&lt;div&gt;</code>）内使用 Markdown 语法时，本地预览和 GitHub Pages 渲染可能不一致。</strong></summary>

<p><strong>原因分析：</strong> kramdown 在 HTML 标签内处理 Markdown 有限制，不同版本的 kramdown 对空白字符和换行的处理可能不同，导致本地 Jekyll 环境和 GitHub Pages 的渲染结果不一致。</p>

<p><strong>解决方案：</strong> 在 HTML 标签内嵌套 Markdown 内容时，优先使用 HTML 标签而不是 Markdown 语法：</p>
<ul>
<li>引用块：使用 <code>&lt;blockquote&gt;</code> 而不是 <code>&gt;</code></li>
<li>列表：使用 <code>&lt;ul&gt;</code>/<code>&lt;ol&gt;</code> 和 <code>&lt;li&gt;</code> 而不是 <code>-</code> 或 <code>1.</code></li>
<li>段落：使用 <code>&lt;p&gt;</code> 包裹段落内容</li>
<li>代码：使用 <code>&lt;code&gt;</code> 或 <code>&lt;pre&gt;&lt;code&gt;</code> 而不是反引号</li>
</ul>

</details>

<details>
<summary><strong>给 AI 的提示词</strong></summary>



<blockquote>
<p><strong>当需要修改 Chirpy 主题的静态资源时：</strong></p>
<pre><code>不能直接修改 Chirpy 主题的静态资源（位于子模块中）。GitHub Pages 构建时子模块可能会未初始化，导致文件缺失。

正确的做法：
1. 将要修改的文件复制到主仓库的相关目录（如 assets/lib-custom/）
2. 更新所有引用路径，从子模块路径改为主仓库路径
3. 确保所有相关文件中的引用都已更新</code></pre>

<hr>

<p><strong>Markdown本地预览和 GitHub Pages 渲染不一致时：</strong></p>
<pre><code>在 HTML 标签（如 &lt;details&gt;、&lt;div&gt;）内编写内容时，请使用 HTML 标签而不是 Markdown 语法，以确保在本地和 GitHub Pages 上渲染一致。例如：
- 使用 &lt;blockquote&gt; 而不是 &gt; 来创建引用块
- 使用 &lt;ul&gt;/&lt;ol&gt; 和 &lt;li&gt; 而不是 - 或 1. 来创建列表
- 使用 &lt;p&gt; 包裹段落内容
- 使用 &lt;code&gt; 或 &lt;pre&gt;&lt;code&gt; 来显示代码</code></pre>
</blockquote>

</details>
