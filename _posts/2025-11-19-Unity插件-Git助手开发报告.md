## 使用说明

### Git 助手

- 首次使用时，请在 Unity 菜单选择 `Tools/Git 助手` 打开窗口。左侧树状图会读取 `git status --porcelain` 并按目录把你的改动分组，方便逐层查看。
- 顶部信息区始终展示当前分支、远端信息与最近一次提交摘要。视觉卡片提供状态概览，你可以随时点击其中的链接跳转到[完整使用教程](https://lyzbcy.github.io/posts/Unity%E6%8F%92%E4%BB%B6-Git%E5%8A%A9%E6%89%8B%E5%BC%80%E5%8F%91%E6%8A%A5%E5%91%8A/)了解细节。
- 操作区域为日常 Git 流程准备了常用按钮，请按需依次点击：
  - **Refresh**：手动刷新状态、分支与日志，确保界面与工作区一致。
  - **Stage All**：执行 `git add -A`，一次性把当前改动全部暂存。若只需部分文件，可在树状图中单独勾选。
  - **Commit**：输入中文或约定格式的提交说明后点击提交，记录当前快照。
  - **Push**：推送至远端，必要时在弹窗中选择目标 remote 与 branch。
  - **Pull**：先打开远端历史列表，挑选目标版本后再执行拉取与同步，避免误合并。
- 右侧“最近提交”卡片会根据窗口高度自动适配显示数量，提交较多时会出现滚动条，确保你能翻阅全部历史。
- 底部日志窗口直接渲染 `git log --graph --oneline`，用图形化分支结构帮助你快速回顾分叉与合并。

### 安装与打包

1. 请把 `Packages/com.fire.gitassistant` 完整放入项目，随后打开 Package Manager 即可通过 UPM 引用。
2. 当你需要对外分发或上架 Asset Store/Package Manager 时，可按场景选择：
   - **UPM Git Tag**：把目录推送到独立仓库并创建 release tag，供用户通过 Git 地址安装。
   - **unitypackage**：在 Unity 中选中 `Packages/com.fire.gitassistant`，执行 `Assets > Export Package...` 导出标准安装包。

### 使用建议

- 请先确认系统已经安装 `git`，并能在命令行直接调用，以免脚本执行失败。
- Unity 编辑器需对项目根目录拥有完整读写权限，尤其是在多人项目或受限目录中。
- 窗口右上角的齿轮按钮提供个性化配置，你可以快速切换界面语言，并设定默认远端与分支，后续操作会沿用该配置。

### 自定义扩展

- 如果需要继续扩展，可参考以下入口：
  - `Editor/GitAssistantWindow.cs`：容纳全部 UI 逻辑，可按业务需求扩展面板或交互。
  - `Editor/GitProcessUtility.cs`：封装 Git 命令调用，你可以在此添加 `git tag`、`git stash` 等额外功能。
  - `Editor/GitTreeView.cs`：负责树状视图呈现，可在此实现右键菜单、单文件暂存等高级操作。
