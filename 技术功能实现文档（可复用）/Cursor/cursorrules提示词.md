# Cursor 规则提示词：统一语言与提交规范

## `.cursorrules` 如何生效

- Cursor 在打开仓库时会自动读取根目录的 `.cursorrules`，并把其中的条目注入到后续对话与自动化任务里。
- 文件内容越具体，代理越容易在回答、写代码、生成提交信息时遵守约束。
- 如果存在多个规则文件（例如子目录中的 `.cursor/rules.json`），Cursor 会按就近原则覆盖或补充；因此保持根级 `.cursorrules` 明确可复用的核心规范很重要。

## 可复用提示词模板（覆盖 `.cursorrules` 第 3-15 行的能力）

将以下片段复制到任意新建的 `.cursorrules` 或其他 Cursor 规则文件中，即可复刻当前仓库的语言与提交规范要求：

```
## 语言偏好
- 所有 AI 生成的内容（包括代码注释、commit message、文档等）都应使用中文
- Commit message 必须使用中文编写
- 代码注释优先使用中文

## Commit Message 规范
- 使用中文描述提交内容
- 格式：简洁明了地说明本次提交的改动
- 示例：
  - "修复：修复了某个 bug"
  - "新增：添加了新功能"
  - "更新：更新了文档"
  - "重构：重构了代码结构"
```

## 使用建议

- 每当需要在子仓库或新项目中保持相同行为，直接将模板段落粘贴到对应 `.cursorrules`，或在对话开始前将其作为系统提示词输入。
- 如需扩展其他约束（例如文件修改流程、测试要求），在模板下方继续增加章节即可，Cursor 会一并遵循。
