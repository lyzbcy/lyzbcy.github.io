# 捞鱼的世界 顶层设计

## 1. 目标摘要

这次重构的核心不是把传统博客换皮，而是把传统博客变成一个可探索的世界入口。

目标来自 `初心与使命.md`：

- 用户打开站点后，首先进入一个类似 `messenger.abeto.co` 的沉浸式开始页。
- 用户可以选择进入世界，也可以进入传统网页。
- 世界里有一个由屏幕组成的标志性建筑，用来浏览文章入口。
- 专栏变成 NPC，比如健身专栏对应健身角色，和他对话能看到健身相关文章。
- 点击文章后不跳出世界，而是在屏幕中央打开一个窗口，展示原 Jekyll/Chirpy 风格网页内容。

## 2. 信息架构

首页应该从现在的“直接重定向到 /world/”升级为一个双入口场景。

- `/`：世界开始页。首屏是 3D/2.5D 沉浸入口，提供两个选择。
- `/world/`：完整可探索世界。
- `/classic/` 或 `/posts/`：传统博客入口。短期可以保留原 Jekyll 首页，长期可以做一个传统网页索引。

推荐首页交互：

- 默认展示世界开始页，背景中可以看到旋转世界球和远处的屏幕塔。
- 主按钮：进入世界。
- 次按钮：进入传统网页版。
- 进入世界时镜头推进到世界球。
- 进入传统网页版时做一个轻量淡出，不破坏原网站阅读体验。

## 3. 世界玩法循环

当前世界的问题是“能走、能点”，但缺少持续探索动机。玩法循环应该变成：

1. 玩家进入世界，出生在主塔附近。
2. 远处能看到多个主题地标和 NPC。
3. 主塔屏幕提供“最新、精选、全部、主题路线”。
4. NPC 提供专栏文章，并带有一两句角色化对话。
5. 点击文章，在世界中间打开阅读窗口。
6. 关闭阅读窗口后回到同一位置继续探索。

最小可玩版本需要补齐：

- 靠近 NPC/屏幕时出现交互提示。
- 对话窗口里文章列表点击后打开内嵌阅读窗口。
- 世界里有 4 到 6 个二级地标，而不是只有主塔和零散树石。
- 地图上给出可见路线，比如发光小径、漂浮路牌、主题光柱。

## 4. 世界内文章窗口

文章窗口是这次体验提升的关键。不要让文章点击直接 `target=_blank` 跳出世界。

推荐实现：

- 新增 `ArticleViewer` UI 管理器。
- NPC 对话和屏幕列表点击文章时调用 `articleViewer.open(post)`。
- 中央弹出一个大窗口，里面用 `iframe` 加载原网页 URL，比如 `/posts/xxx/`。
- 窗口顶部保留世界风格标题栏：文章标题、分类、日期、关闭按钮、在新标签打开按钮。
- iframe 内部保留原 Jekyll/Chirpy 页面，这样“原网页内容”和原有 CSS/JS 都能继续工作。

需要注意：

- 同域 iframe 可以直接加载本地文章，部署到 GitHub Pages 也成立。
- 如果文章里有 `target=_blank`，保留默认行为。
- 小屏幕上阅读窗口应接近全屏。
- 打开文章时暂停玩家移动，关闭后恢复。

## 5. 建筑与地图扩展

当前主塔是正确方向，但周围内容太少。地图应该从“一个塔”扩展成“一个主题群岛/星球街区”。

推荐建筑分层：

- 中心：屏幕塔，承担站点总入口。
- 健身区：训练馆、计分牌、力量图标，对应健身 NPC。
- 技术区：服务器塔、蓝色工作站、工具屏幕，对应 Unity/工具/学习。
- 生活区：小屋、留言牌、暖色灯，对应生活/写作/星星布丁。
- 旅行/美食区：小摊、路标、餐车、地图牌，对应旅游/美食。
- 未来规划区：办公楼、公告屏、路线板，对应就业/社会体验。

短期不用一次性引入复杂外部模型。可以先用代码生成低多边形建筑，保证风格统一；第二阶段再引入 CC0 glTF。

## 6. 资源策略

优先使用 CC0/公共领域资产，避免授权负担。

建议资源来源：

- Kenney City Kit / Modular Buildings：低多边形建筑，CC0，适合补城市与主题建筑。
- Quaternius：大量免费低多边形 3D 模型，CC0，适合补奇幻/地标/装饰。
- Poly Haven：天空 HDRI 和环境资源，CC0，可作为天空球参考或后续贴图来源。
- Kenney Animated Characters：如需要从 2D NPC 升级到 3D NPC，可用它做统一角色风格。

落地原则：

- 不混用写实模型和卡通模型。
- 优先 glTF，避免 OBJ/FBX 带来的材质处理成本。
- 每个外部资源进入项目前记录来源、许可证和下载日期。
- 对 GitHub Pages 要控制体积，首批资源建议总量不超过 5MB。

## 7. NPC 美术方向

当前 NPC 是 Canvas 画的小人，功能清楚，但质感不够。

两条路线：

- 路线 A：继续 2D 精修。用更高分辨率 Canvas 绘制、加入阴影、轮廓光、角色姿态和主题道具。优点是轻、可控、加载快。
- 路线 B：改为低多边形 3D 角色。用 Kenney/Quaternius 风格模型，加入 idle 动画。优点是更像游戏世界，但工程量更大。

推荐先走路线 A：

- 保持 messenger 风格的轻量感。
- 每个 NPC 有主题剪影，不追求复杂骨骼动画。
- 角色旁边加小型主题建筑/道具，让识别度来自“角色 + 场景”。

## 8. 天空球与世界球

当前天空是基础渐变星空，地表是噪声球。下一步应该提升“世界氛围”。

天空球改造：

- 增加多层星云渐变。
- 增加慢速流动的云带/极光带。
- 星星分成远景微星、近景亮星、偶发流星。
- 保留 Shader，不急于上大 HDRI，避免首屏资源太重。

世界球改造：

- 地形颜色更分区：主塔广场、森林带、沙岸、岩石区。
- 路径更明显：发光道路连接主塔和 NPC。
- 添加边缘轮廓光，让星球在开始页更有识别度。
- 加入更多可读的地标，而不是随机树石。

## 9. 开始页交互

`messenger.abeto.co` 给人的重点不是“控件很多”，而是入口有节奏、画面轻、过渡顺。

开始页应该这样收敛：

- 首屏少文案，只保留世界名称、一句说明、两个入口。
- 背景直接展示可旋转的世界球，而不是静态说明卡。
- 鼠标移动时世界球和 UI 有轻微视差。
- 点击“进入世界”后镜头推进，UI 退场，玩家接管。
- 点击“传统网页版”后进入原 Jekyll 站。
- 保留轻量 loading 进度，但不要让 loading 成为视觉主体。

## 10. 分期实现

第一阶段：阅读闭环

- 新增中央文章阅读窗口。
- 修改 NPC 对话和屏幕列表：文章点击打开窗口，而不是新标签。
- 打开阅读窗口时暂停玩家移动和交互。
- 首页增加“进入世界 / 传统网页版”的双入口。

第二阶段：可玩性

- 增加 4 到 6 个主题地标。
- 增加发光路径和主题区域。
- NPC 增加靠近提示、短对白、主题道具。
- 主塔屏幕按“最新 / 精选 / 分类路线”重新编排。

第三阶段：视觉质量

- 天空球升级为星云、极光、流星的 Shader。
- 世界球加入区域色彩、轮廓光和更强路径设计。
- NPC 进行 2D 精修或统一替换为低多边形角色。
- 引入少量 CC0 glTF 建筑，建立资源清单。

第四阶段：首页接管

- `/` 正式成为世界开始页。
- 传统网页版保留可访问入口。
- 做移动端交互适配。
- 做性能预算和首屏加载验证。

## 11. 下一步建议

下一次实现应优先做第一阶段。理由很简单：文章不能在世界里读，世界就只是一个菜单；文章能在世界中间打开，整个设计才真正闭环。
